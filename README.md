# 프로젝트명

건설 도면 탐색 플랫폼

## 실행 방법

npm install
npm run dev

## 기술 스택

- React 19.2.0
- TypeScript 5.9.3
- Vite 7.3.1 (Build Tool)
- ESLint + Prettier (Code Quality)
- Airbnb ESLint Style Guide

## 구현 기능

- [O] 도면 탐색: 사용자가 원하는 도면을 찾을 수 있어야 함
- [O] 도면 표시: 선택한 도면 이미지를 화면에 표시
- [O] 컨텍스트 인식: 현재 어떤 도면을 보고 있는지 사용자가 알 수 있어야 함

## 설계 방식

- **데이터 분석**: 
  - `metadata.json`은 도면(Drawing), 공종(Discipline), 영역(Region), 그리고 개정(Revision)이 계층적으로 얽힌 복합적인 구조로 되어 있습니다. 특히 `imageTransform`과 `vertices` 같은 기하학적 데이터가 포함되어 있어, 도면 간의 상대적 위치나 변형(scale, rotation) 정보를 담고 있음을 이해했습니다.
  - **더 나은 데이터 표현 방법**: 현재의 중첩된 구조는 프론트엔드에서 직접 순회하며 필터링하기에 복잡하므로, 데이터 제공 시 도면-공종-개정 정보를 **Flatten(평탄화)**하여 제공하거나, GraphQL과 같이 필요한 계층만 쿼리할 수 있는 방식이 유지보수와 성능 면에서 유리할 것이라 판단했습니다.

- **접근 방식**: 
  1. **데이터 모델링**: `metadata.json`을 분석하여 TypeScript 인터페이스를 정의하고 데이터의 계층 구조를 명확히 했습니다.
  2. **데이터 가공 (Normalization)**: 복잡한 중첩 데이터를 UI에서 사용하기 쉬운 형태(`AppDrawing[]`)로 변환하는 커스텀 훅(`useMetadata`)을 개발했습니다.
  3. **레이아웃 구성**: 사이드바(필터 및 목록)와 메인 뷰어 영역을 구분하는 기본 골격을 잡았습니다.
  4. **기능 구현**: 공종별 필터링, 도면 선택, 그리고 비교 모드 기능을 순차적으로 구현했습니다.
  5. **예외 처리**: 한글 파일명의 유니코드 정규화(NFD) 문제를 해결하여 이미지 로딩의 안정성을 확보했습니다.

- **UI 설계 결정**:
  - **레이아웃**: 좌측 사이드바 + 우측 메인 뷰어 패턴을 채택했습니다. 이는 수많은 도면 목록을 탐색하면서도 메인 콘텐츠(도면)에 집중할 수 있는 직관적인 구성입니다. 상단 헤더를 고정하여 현재 컨텍스트를 항상 확인할 수 있게 했습니다.
  - **대안 및 장단점**: 모달(Modal) 방식도 고려했으나, 여러 도면을 빈번하게 전환하며 확인해야 하는 업무 특성상 화면 전환 비용이 낮고 탐색이 자유로운 사이드바 방식이 최선이라 판단했습니다.

- **기술 선택**: 
  - **React & TypeScript**: 컴포넌트 기반의 UI 개발과 엄격한 타입 체크를 통해 데이터 구조의 복잡성을 제어하고자 선택했습니다.
  - **Vanilla CSS (Inline Styles)**: 별도의 라이브러리 의존성을 최소화하고 핵심 로직에 집중하며, 스타일 정의를 컴포넌트 내로 한정하여 빠른 프로토타이핑을 진행했습니다.
  - **State Management**: React 내장 Hooks(`useState`, `useMemo`)만을 활용하여 전역 상태 라이브러리 없이도 효율적인 데이터 흐름을 구현했습니다.

- **어려웠던 점 및 개선 방안**: 
  - **어려웠던 점**: macOS와 Windows 간의 한글 파일명 자소 분리(NFD) 문제로 인해 이미지 경로가 일치하지 않는 현상이 있었습니다. 이를 `normalize('NFD')`를 통해 해결했습니다.
  - **개선 방안**: 도면 뷰어에 **Zoom/Pan 기능**을 추가하여 세부 내용을 확인할 수 있게 하고, 비교 모드에서 두 도면을 겹쳐서 차이를 보여주는 **Overlay/Slider 기능**을 구현하고 싶습니다. 또한, 메타데이터의 좌표 정보를 활용해 도면 내 특정 영역(Region)을 인터랙티브하게 강조하는 기능을 추가하고 싶습니다.
